# 示例技能

此技能演示了 Claude Code 插件技能的结构和格式。

## 概述

技能是由模型调用的功能，Claude 根据任务上下文自主使用。与命令（用户调用）或代理（由 Claude 生成）不同，技能提供 Claude 融入其响应的上下文指导。

## 此技能适用的场景

当用户的请求涉及以下情况时，此技能激活：
- 创建或理解插件技能
- 需要技能模板或参考
- 技能开发模式

## 技能结构

### 必需文件

```
skills/
└── skill-name/
    └── SKILL.md          # 主要技能定义（必需）
```

### 可选支持文件

```
skills/
└── skill-name/
    ├── SKILL.md          # 主要技能定义
    ├── README.md         # 额外文档
    ├── references/       # 参考材料
    │   └── patterns.md
    ├── examples/         # 示例文件
    │   └── sample.md
    └── scripts/          # 辅助脚本
        └── helper.sh
```

## 前置元数据选项

技能支持以下前置元数据字段：

- **name**（必需）：技能标识符
- **description**（必需）：触发条件 - 描述 Claude 应何时使用此技能
- **version**（可选）：语义版本号
- **license**（可选）：许可证信息或参考

## 编写有效的描述

description 字段至关重要 - 它告诉 Claude 何时调用技能。

**良好的描述模式：**
```yaml
description: 当用户要求"特定短语"、"另一个短语"，提及"关键字"或讨论主题区域时，应使用此技能。
```

**包含：**
- 用户可能说的具体触发短语
- 表示相关性的关键字
- 技能涵盖的主题区域

## 技能内容指南

1. **明确目的**：说明技能帮助完成什么
2. **何时使用**：定义激活条件
3. **结构化指导**：逻辑地组织信息
4. **可操作的指令**：提供具体步骤
5. **示例**：在有帮助时包含实用示例

## 最佳实践

- 保持技能专注于单一领域
- 编写明确指示何时激活的描述
- 对于复杂技能，在子目录中包含参考材料
- 测试技能是否对预期查询激活
- 避免与其他技能的触发条件重叠
