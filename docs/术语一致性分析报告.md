### 3.2 需要统一修正的术语

#### 修正项 1: Agent 的翻译

**现状**：在不同文档中使用了 "Agent"、"代理"、"智能体" 三种翻译。

**建议**：
- 作为产品功能名称时使用 "Agent"（如：Agent Teams）
- 作为通用技术概念时使用 "代理"（如：子代理）
- 避免使用 "智能体"，容易与 AI 智能体混淆

**需要修改的文档**：无（当前使用基本合理）

#### 修正项 2: Pull Request 的翻译

**现状**：部分文档使用 "PR"，部分使用 "Pull Request"，部分使用 "拉取请求"。

**建议**：统一使用 "PR"（更简洁，开发者通用）

**需要修改的文档**：
- 《工程化实践篇》：将 "Pull Request" 统一改为 "PR"

#### 修正项 3: API key 的翻译

**现状**：存在 "API 密钥"、"API Key"、"API 密钥" 三种形式。

**建议**：统一使用 "API 密钥"（中文更规范）

**需要修改的文档**：
- 《配置文件完全指南》：将 "API Key" 改为 "API 密钥"
- 《插件市场完全指南》：统一使用 "API 密钥"

#### 修正项 4: Repository 的翻译

**现状**：存在 "仓库" 和 "代码仓库" 两种翻译。

**建议**：统一使用 "仓库"（更简洁）

**需要修改的文档**：无需修改（当前主要使用 "仓库"）

#### 修正项 5: Token 的翻译

**现状**：存在 "Token" 和 "令牌" 两种翻译。

**建议**：技术上下文统一使用 "Token"（如：API Token、上下文 Token）

**需要修改的文档**：无需修改（当前主要使用 "Token"）

## 五、术语翻译规范建议

### 5.1 总体原则

1. **保留英文的技术术语**：SDK、MCP、API、CLI、JSON 等
2. **品牌名称保留英文**：Claude Code、Anthropic、GitHub、GitLab 等
3. **文件名保留英文**：settings.json、CLAUDE.md 等
4. **命令保留英文**：所有以 / 开头的命令
5. **编程语言保留英文**：TypeScript、Python、JavaScript 等

### 5.2 翻译优先级

| 优先级 | 处理方式 | 示例 |
|--------|---------|------|
| **高** | 保持英文 | SDK、MCP、Claude Code |
| **中** | 中文翻译 | 权限、插件、代理 |
| **低** | 根据上下文 | Agent（Agent Teams vs 子代理） |

### 5.3 特殊情况处理

1. **首次出现时加注英文**：
   - 示例：MCP（Model Context Protocol，模型上下文协议）

2. **命令行代码块保留英文**：
   - 所有命令、参数、输出均使用英文

3. **专有名词大小写**：
   - GitHub、GitLab、Bitbucket（正确）
   - github、gitlab（错误）
