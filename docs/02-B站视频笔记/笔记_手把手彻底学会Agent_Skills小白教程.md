## 重要概念与术语

| 术语 | 解释 |
|------|------|
| **Agent Skills** | Claude 的模块化扩展包，为 Claude 提供特定领域的专业知识、工作流程和工具集成能力 |
| **SKILL.md** | 每个技能的核心文件，包含 YAML 前置元数据（name、description）和 Markdown 格式的使用指南 |
| **Frontmatter** | SKILL.md 顶部的 YAML 元数据区域，包含 `name` 和 `description` 字段，是技能触发的关键 |
| **scripts/** | 存放可执行脚本的目录（Python/Bash等），用于需要确定性可靠执行的任务 |
| **references/** | 存放参考文档的目录，Claude 在需要时按需加载，避免占用过多上下文窗口 |
| **assets/** | 存放输出资产的目录（模板、图片、字体等），不会被加载到上下文中，而是用于最终输出 |
| **渐进式披露** | Skill 的三级加载机制：元数据（始终加载）→ SKILL.md 正文（触发时加载）→ 捆绑资源（按需加载） |
| **秋芝技能生成器** | UP主开发的辅助工具，帮助用户快速生成 Skill 模板和结构 |

## Skill 的目录结构

```
skill-name/
├── SKILL.md          # 必需 - 技能主文件
├── scripts/          # 可选 - 可执行脚本
├── references/       # 可选 - 参考文档
└── assets/           # 可选 - 输出资产（模板、图片等）
```

## 制作 Skill 的基本流程

### 步骤 1：理解需求
- 明确技能要解决什么问题
- 收集具体的使用场景和示例

### 步骤 2：规划技能内容
- 分析每个使用场景需要哪些可复用资源
- 确定需要哪些脚本、参考文档和资产

### 步骤 3：初始化技能
- 使用 `init_skill.py` 脚本创建技能模板目录
- 自动生成 SKILL.md 模板和示例文件

### 步骤 4：编辑技能
- 编写 SKILL.md 的 frontmatter（name + description）
- 编写技能使用指南
- 实现脚本、参考文档和资产文件

### 步骤 5：打包技能
- 使用 `package_skill.py` 打包为 `.skill` 文件
- 自动验证技能结构和内容

### 步骤 6：测试与迭代
- 在实际任务中使用技能
- 根据使用效果持续优化

## SKILL.md 编写要点

1. **description 是关键**：这是技能的触发机制，必须清晰描述技能的功能和使用场景
2. **正文要精简**：上下文窗口是公共资源，只写 Claude 不知道的信息
3. **善用渐进式披露**：核心内容放 SKILL.md，详细内容放 references/
4. **SKILL.md 正文控制在 500 行以内**
